# Capacitor 8.2.0 iOS Setup Guide

This guide ensures your Xcode project builds and runs correctly with Capacitor 8.2.0, Camera 8.2.0, and Geolocation 8.2.0.

## Prerequisites

- Xcode 14.0+
- iOS 13.0+
- Node.js 16+
- Capacitor CLI: `npm install -g @capacitor/cli@latest`

## Project Structure

```
project-root/
├── src/                           # React/TypeScript source
├── dist/                          # Built web assets (output from `npm run build`)
├── capacitor.config.ts            # Capacitor configuration
├── ios/                           # Native iOS project (generated by Capacitor)
│   ├── Podfile                    # CocoaPods dependencies
│   ├── Asciende.xcworkspace       # Xcode workspace
│   ├── App/
│   │   ├── Podfile                # Generated by Capacitor
│   │   └── Info.plist             # App configuration
```

## Setup Steps

### 1. Install Dependencies

```bash
npm install
npm install -g @capacitor/cli
```

### 2. Build the Web Project

```bash
npm run build
```

This creates the `dist/` folder with all web assets needed for the app.

### 3. Initialize or Sync iOS Project

**If no iOS project exists:**

```bash
npx cap add ios
```

**If iOS project exists (update it):**

```bash
npx cap sync ios
```

This command:
- Copies web assets to iOS
- Updates plugin dependencies
- Configures native permissions

### 4. Open in Xcode

```bash
npx cap open ios
```

Or manually:
```bash
open ios/Asciende.xcworkspace
```

**IMPORTANT**: Always open `.xcworkspace`, NOT `.xcodeproj`

### 5. Configure Info.plist

Xcode should have created/updated `ios/App/Info.plist` with required permissions. Verify these keys exist:

```xml
<key>NSLocationWhenInUseUsageDescription</key>
<string>Asciende needs your location to record GPS activities (running, cycling, swimming)</string>

<key>NSCameraUsageDescription</key>
<string>Asciende needs camera access for video analysis and photo capture</string>

<key>NSLocationAlwaysAndWhenInUseUsageDescription</key>
<string>Asciende needs your location for background activity tracking</string>
```

### 6. Build in Xcode

1. Select scheme: **Asciende** (or App)
2. Select device/simulator
3. Product → Build (Cmd+B)
4. Product → Run (Cmd+R)

## Plugin Configuration

### Camera Plugin

**Location**: `capacitor.config.ts`

```typescript
plugins: {
  Camera: {
    permissions: ['camera'],
  },
}
```

**iOS Usage**:
- Permission request handled automatically by Capacitor 8
- `@capacitor/camera 8.2.0` provides:
  - `Camera.requestPermissions({ permissions: ['camera'] })`
  - Returns `{ camera: 'granted' | 'denied' | 'prompt' }`

### Geolocation Plugin

**Location**: `capacitor.config.ts`

```typescript
plugins: {
  Geolocation: {
    permissions: ['location'],
  },
}
```

**iOS Usage**:
- Permission request handled by Capacitor 8
- `@capacitor/geolocation 8.2.0` provides:
  - `Geolocation.requestPermissions({ permissions: ['geolocation'] })`
  - `Geolocation.watchPosition(options, callback, errorCallback)`
  - `Geolocation.clearWatch({ id: watchId })`

## Common Build Errors & Solutions

### Error: "Module not found: @capacitor/geolocation"

**Solution**: Reinstall dependencies and sync

```bash
rm -rf node_modules package-lock.json
npm install
npx cap sync ios
```

### Error: "Swift compilation failed" or "AppDelegate mismatch"

**Solution**: Clean build

```bash
npx cap sync ios
cd ios
rm -rf Pods Podfile.lock
pod install --repo-update
cd ..
```

Then in Xcode: Product → Clean Build Folder (Shift+Cmd+K)

### Error: "Permission denied" at runtime

**Solution**: Verify Info.plist permissions are present. Rebuild and reinstall app on device.

### Error: "watchPosition not working on iOS"

**Solution**: Ensure:
1. Location permission is granted in device settings
2. App has NSLocationWhenInUseUsageDescription in Info.plist
3. Geolocation plugin is initialized before calling watchPosition

## TypeScript Definitions

TypeScript types are automatically provided by:
- `@capacitor/core` (Capacitor API types)
- `@capacitor/camera` (Camera plugin types)
- `@capacitor/geolocation` (Geolocation plugin types)

All plugin imports are type-safe:

```typescript
import { Camera } from '@capacitor/camera';
import { Geolocation } from '@capacitor/geolocation';
```

## Debugging

### Enable Verbose Logging

In Xcode, add launch argument:
- Product → Scheme → Edit Scheme → Run
- Arguments Passed On Launch: `-com.apple.CoreData.ConcurrencyDebug 1`

### Console Output

In Xcode console, search for:
- `Capacitor` - Shows Capacitor initialization
- `GeolocationPlugin` - GPS plugin debug messages
- `CameraPlugin` - Camera plugin debug messages

## Testing on Device

1. Connect iPhone via USB
2. Xcode → Devices and Simulators
3. Trust the computer on iPhone
4. Select device in Xcode
5. Product → Run (Cmd+R)

To test location/camera:
- iOS Settings → Privacy → Location/Camera → Asciende → Allow

## Build for Release

```bash
# 1. Update version in capacitor.config.ts
# 2. Build web
npm run build

# 3. Sync iOS
npx cap sync ios

# 4. In Xcode, select "Generic iOS Device"
# 5. Product → Archive
# 6. Follow signing and distribution steps
```

## Troubleshooting Checklist

- [ ] `npm run build` succeeds
- [ ] `npx cap sync ios` completes without errors
- [ ] `.xcworkspace` file opens in Xcode (not `.xcodeproj`)
- [ ] Info.plist has all required permission strings
- [ ] CocoaPods dependencies installed (`pod install`)
- [ ] Xcode build succeeds (Cmd+B)
- [ ] App launches on simulator/device (Cmd+R)
- [ ] Location permission request appears on first run
- [ ] Camera permission request appears when needed
